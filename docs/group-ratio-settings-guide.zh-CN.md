# new-api 分组功能学习文档

> 适用页面：**系统设置 → 分组与模型定价设置 → 分组相关设置**
>
> 本文重点解释你截图里的 5 块配置：
> - 分组倍率
> - 用户可选分组
> - 分组特殊倍率
> - 分组特殊可用分组
> - 自动分组 auto

---

## 1. 先记住两个概念

### 1.1 用户分组

用户账号本身的分组，存的是 `user.Group`。

例如：

- `default`
- `vip`
- `Codex`

它表示“这个用户属于哪一类人”。

---

### 1.2 令牌分组

用户创建令牌时，令牌实际使用的分组。

例如：

- `default`
- `CC专用`
- `CC第三方可用`
- `Codex`
- `auto`

它表示“这个 token / 请求走哪组渠道与倍率规则”。

---

## 2. 这页配置到底控制什么

这页本质上是在定义：

> **用户分组(user.Group)**
> → **能看到/能选择哪些令牌分组**
> → **使用这些令牌分组时按什么倍率计费**

也就是说：

- **用户可选分组**：决定默认给用户显示哪些令牌分组
- **分组特殊可用分组**：决定某类用户额外加哪些、删哪些令牌分组
- **分组倍率**：定义每个令牌分组的基础倍率
- **分组特殊倍率**：按用户分组覆盖某个令牌分组的倍率
- **自动分组 auto**：用户选择 `auto` 时按顺序尝试哪些分组

---

## 3. 你截图里的配置逐行解释

以下按截图中的内容解释。

---

### 3.1 分组倍率

```json
{
  "Codex": 15,
  "CC第三方可用": 6,
  "CC专用": 5,
  "default": 1.5
}
```

#### 含义

这定义了每个**令牌分组**的基础倍率：

- `Codex = 15`
- `CC第三方可用 = 6`
- `CC专用 = 5`
- `default = 1.5`

#### 人话理解

如果某个请求最终使用了这个分组，就先按这个倍率计费。

例如：

- 请求走 `Codex` 分组 → 基础倍率按 `15`
- 请求走 `default` 分组 → 基础倍率按 `1.5`

除非后面的**分组特殊倍率**把它覆盖掉。

---

### 3.2 用户可选分组

```json
{
  "Codex": "Codex分组",
  "CC专用": "CC专用分组，只能用在CC终端，请勿在其他地方使用",
  "CC第三方可用": "第三方（openclaw等）可用",
  "default": "默认分组，已弃用勿选"
}
```

#### 含义

这是**默认情况下，用户能看到/能选择的令牌分组列表**。

#### 逐行解释

- `Codex`
  - 用户可以看到并选择 `Codex`
  - 展示说明：`Codex分组`

- `CC专用`
  - 用户可以看到并选择 `CC专用`
  - 展示说明：`CC专用分组，只能用在CC终端，请勿在其他地方使用`

- `CC第三方可用`
  - 用户可以看到并选择 `CC第三方可用`
  - 展示说明：`第三方（openclaw等）可用`

- `default`
  - 用户可以看到并选择 `default`
  - 展示说明：`默认分组，已弃用勿选`
  - 这里只是提示文案，**不会自动禁止选择**

#### 当前效果

按这份配置，默认用户会看到 4 个令牌分组：

- `Codex`
- `CC专用`
- `CC第三方可用`
- `default`

---

### 3.3 分组特殊倍率

```json
{
  "CC第三方可用": {
    "edit_this": 0.9
  }
}
```

#### 结构怎么读

它的结构是：

```json
{
  "用户分组": {
    "令牌分组": 特殊倍率
  }
}
```

#### 所以你这条配置真正的意思是

- 如果某个用户的 **用户分组** 是 `CC第三方可用`
- 并且他使用的 **令牌分组** 是 `edit_this`
- 那么倍率按 `0.9`

#### 但你这条大概率没有实际效果

因为你正式分组里并没有 `edit_this` 这个令牌分组。

所以它更像是：

- 示例值
- 占位值
- 测试残留

#### 如果你真正想表达的是

“用户分组 = `CC第三方可用` 的用户，在使用 `default` 时按 0.9 倍率”

那应该写成：

```json
{
  "CC第三方可用": {
    "default": 0.9
  }
}
```

---

### 3.4 分组特殊可用分组

```json
{
  "vip": {
    "-:remove_1": "vip_removed_group_1",
    "append_1": "vip_special_group_1"
  }
}
```

#### 结构怎么读

它的结构是：

```json
{
  "用户分组": {
    "规则": "描述"
  }
}
```

#### 外层 `vip` 是什么意思

表示：

> 如果某个用户的 `user.Group = vip`
> 就对他的“可选令牌分组列表”做特殊处理

#### 里面两条规则分别是什么意思

##### 1）`"-:remove_1": "vip_removed_group_1"`

- `-:` 前缀表示 **移除**
- 如果这个用户默认本来能看到 `remove_1`
- 那就把 `remove_1` 从可选分组里删掉

##### 2）`"append_1": "vip_special_group_1"`

- 没有 `-:` 前缀
- 当前后端会把它当成**直接追加一个分组**
- 也就是给 `vip` 用户额外增加一个可选分组 `append_1`
- 显示描述为 `vip_special_group_1`

#### 这条配置为什么看起来也不像正式业务配置

因为你前面的正式令牌分组里没有：

- `remove_1`
- `append_1`

而且截图里也不一定真的有用户分组 `vip` 在使用。

所以这条也大概率是：

- 示例配置
- 模板配置
- 尚未正式启用

---

### 3.5 自动分组 auto

```json
[
  "default"
]
```

#### 含义

当令牌分组选 `auto` 时，系统会按这个顺序尝试分组。

你这里目前只有一个值：

- `default`

#### 当前效果

如果用户令牌设置为 `auto`，那系统现在基本等于：

> `auto = default`

也就是：

1. 先尝试 `default`
2. 没有第二个备选分组

## 5. 最容易搞混的点

### 5.1 “特殊可用分组”不是按单个账号配

它是按：

> **用户分组(user.Group)**

来配的。

也就是说，不是直接写：

- 张三能看到 A
- 李四能看到 B

而是写：

- `vip` 用户能看到 A
- `Codex` 用户能看到 B

如果你想只给单个用户特殊权限，常见做法是：

- 给这个用户单独建一个用户分组
- 再按这个组配置

---

### 5.2 “特殊倍率”不是改分组本身的倍率

它是：

> **某类用户在使用某个分组时，用一个覆盖倍率**

不是全局修改该分组的基础倍率。

---

### 5.3 `auto` 不是“随便挑一个组”

`auto` 只会在：

- 当前用户有权限使用的分组里
- 按你配置的顺序

进行尝试。

## 6. 一个更正式、可落地的示例

如果你以后想把“特殊规则”真正用起来，可以参考这种写法。

### 分组倍率

```json
{
  "Codex": 15,
  "CC第三方可用": 6,
  "CC专用": 5,
  "default": 1.5
}
```

### 用户可选分组

```json
{
  "CC专用": "CC专用分组",
  "CC第三方可用": "第三方可用",
  "default": "默认分组"
}
```

### 分组特殊可用分组

```json
{
  "Codex": {
    "-:CC专用": "remove",
    "-:CC第三方可用": "remove",
    "+:Codex": "Codex 专线"
  }
}
```

### 分组特殊倍率

```json
{
  "Codex": {
    "default": 6,
    "Codex": 12
  }
}
```

### 自动分组

```json
[
  "Codex",
  "CC第三方可用",
  "default"
]
```

### 这套配置的业务效果

- 普通用户默认看到：
  - `CC专用`
  - `CC第三方可用`
  - `default`

- `Codex` 用户看到：
  - `default`
  - `Codex`

- `Codex` 用户用 `default` 时倍率按 `6`
- `Codex` 用户用 `Codex` 时倍率按 `12`
- 用户选 `auto` 时会优先按 `Codex → CC第三方可用 → default` 尝试

---

## 7. 一句话总结

你这页配置可以总结成一句话：

> **用户分组决定用户能看到哪些令牌分组，以及用户使用这些令牌分组时按什么倍率计费。**

而你截图里的当前配置，主要是：

- 定义了 4 个正式令牌分组
- 默认都对用户开放
- `auto` 目前只会走 `default`
- “特殊倍率”和“特殊可用分组”看起来还是示例值，尚未真正对正式分组生效
